4 — Coding for Non-Coders

Rashmi Rama Sushil · April 2023 · 5 min read

I had no coding background. My undergraduate was in mechanical engineering, and my IIT Madras research involved ML tools I'd learned along the way — but not DSA, not competitive programming, not anything that would help in a placement test.

Here's exactly what I did, and what I'd recommend to anyone in the same position.

Language: Python

I chose Python for three reasons: I'd used it briefly in a college course so it wasn't completely foreign, it's widely accepted in placement tests, and it's readable — as a non-programmer, the syntax didn't feel hostile. If you have a similar background, Python is the right choice.

The starting point: DSA sheet by Shraddha and Aman

I used the DSA sheet by Shraddha Khapra and Aman Dhattarwal as my structure. It organises problems by topic and difficulty (easy, medium, hard). I focused on easy and medium questions, documented the key logic in a notebook, and marked questions I had to attempt more than once with asterisks — those came back for revision.

Topics covered, in order:

  1. Arrays
  2. Strings
  3. 2D Arrays
  4. Searching and Sorting
  5. Backtracking
  6. Dynamic Programming
  7. Heaps and Hashing
  8. Linked Lists
  9. Binary Trees and Binary Search Trees
  10. Stacks and Queues
  11. Graphs (basics only)

How to work through each topic

The sheet directs you to LeetCode and GFG. For each problem, attempt it with your current knowledge first. Set a time limit: no more than 30–40 minutes on an easy question, no more than an hour on medium. After that, look up the solution. The goal isn't to solve every problem independently — it's to understand the patterns and recognise them the next time you see them.

YouTube channels that actually helped:

  • NeetCode — best overall. Clear explanations with working solutions.
  • Love Babbar — best for recursion, backtracking, and BSTs.
  • Take U Forward — best for stacks and queues.
  • Aditya Verma — best for Dynamic Programming.

Practice platforms: LeetCode (primary), GFG, HackerRank, HackerEarth (for mock tests in the format used by placement assessments).

Beyond DSA

SQL and OOP concepts also come up in placement tests, especially for analytics and quant roles. I covered basic SQL queries, joins, and aggregations, and revised OOP concepts (classes, inheritance, polymorphism) through YouTube explanations.

The one thing that matters most

Revision is the key. Seeing similar questions repeatedly until you start recognising the pattern — that's what builds confidence in a timed test. Don't worry about covering every topic perfectly. Get started, cover the core topics, revise the ones you struggled with, and trust that it compounds.

Just get started. I promise you, it will work out.
